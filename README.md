# solaris_remastred_safety

This project is a small Windows GDI demo built around `main.cpp`.
It is designed as a fullscreen visual/audio effect loop, not as a real malware sample.

## Project Structure

- `main.cpp` - primary executable, fullscreen desktop effects, phase-based audio, cursor jitter, and GDI drawing
- `main.exe`, `main.obj`, `main.pdb`, `main.ilk`, `vc140.pdb` - build outputs and debug artifacts

## `main.cpp` Overview

`main.cpp` is organized into a few clear stages:

1. Header and library setup
2. Audio buffer generation
3. Program start and confirmation dialog
4. Fullscreen effect loop
5. Cleanup on exit

## Top-Level Pieces

### Includes and linking

`main.cpp` uses:

- `<windows.h>` for Win32/GDI APIs
- `<cmath>` for sine-based color motion
- `<ctime>` for seeding random values

It links against:

- `user32.lib`
- `gdi32.lib`
- `winmm.lib`

### Constants

- `SAMPLE_RATE` - audio sample rate used by the generated buffer
- `BUFFER_SIZE` - size of the generated byte buffer
- `PHASE_DURATION` - time window, in milliseconds, for each visual/audio phase

### Data types

- `RGBQUAD_CUSTOM` - helper union for direct pixel manipulation
- `BouncingCircle` - small state object for the moving circles drawn on screen

### Audio generation

`FillBeatBuffer(unsigned char* buffer, int size, int phase)` fills the sound buffer with bytebeat-style formulas.

Behavior:

- switches on `phase % 16`
- computes a different expression for each pattern
- stores the result into the buffer one byte at a time

This is what drives the changing background audio when phases change.

## `main()` Flow

### 1. Startup

- seeds the random generator with `time(0)`
- shows a confirmation `MessageBoxW`
- exits immediately if the user does not choose Yes

### 2. Screen and animation setup

- reads screen width and height with `GetSystemMetrics`
- initializes a `color_timer`
- tracks the previous phase with `last_phase`
- creates six bouncing circles with random position, velocity, and size

### 3. Audio setup

- opens the default wave output device with `waveOutOpen`
- allocates a large buffer
- fills the buffer for the first phase
- prepares a looping `WAVEHDR`
- writes the buffer to the audio device

### 4. Main loop

The loop runs until Escape is held down.

Each frame:

- gets the desktop device context with `GetDC(0)`
- picks a random number `r`
- computes elapsed time since launch
- converts elapsed time to the current phase
- refreshes the audio buffer when the phase changes
- computes a smooth RGB color with sine waves
- updates and draws six bouncing circles
- sometimes nudges the cursor by a random offset
- applies the current phase effect
- releases the device context and sleeps briefly

### 5. Exit cleanup

- resets and unprepares the wave header
- closes the audio device
- deletes the audio buffer
- invalidates the desktop area

## Phase Map in `main.cpp`

`PHASE_DURATION` controls how long each phase lasts.

- Phase 0 - desktop stretch effect
- Phase 1 - random rectangles and ellipses
- Phase 2 - vertical strip shifting with `BitBlt`
- Phase 3 - random `SOLARIS` text drawing
- Phase 4 - sine-based horizontal wave distortion
- Phase 5 - vertical melt-style `BitBlt`
- Phase 6 - XOR-style pixel manipulation using a DIB section
- Phase 7 - full-screen color inversion
- Phase 8 - full-screen patterned fill
- Phase 9 - narrow horizontal strip copy
- Phase 10 - left/right screen wrap distortion
- Phase 11 - widened stretch distortion
- Phase 12 - clipped elliptical inversion
- Phase 13 and above - final phase with `MATVIJ` text, inverted blits, and melt effects

## Notes

- The program is interactive only at launch and exit; the main effect loop is otherwise automatic.
- Escape stops the loop.
- The visual behavior is intentionally chaotic, but the code is still a single-file Win32 demo centered on `main.cpp`.

## VS Code

- Open this folder in VS Code.
- Install the recommended `C/C++` extension if prompted.
- The build script uses `D:\MSVC\Product\VC\Auxiliary\Build\vcvars64.bat` by default.
- Use `Terminal > Run Build Task` or `Ctrl+Shift+B` to compile.
- Press `F5` or use the `Run solaris` launch config to build and start the executable from VS Code.

## Visual Studio

- Open `solaris.sln` in Visual Studio.
- The solution includes `Release|x64` and `Debug|x64`.
- `Release|x64` is listed first so it opens as the default configuration.
- The project file is `solaris.vcxproj`, and it builds `main.cpp` as a Windows subsystem application.
